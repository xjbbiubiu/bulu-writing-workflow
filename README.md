# bulu-writing-workflow

`bulu-writing-workflow` 是一个为公众号「布鲁成长手记」设计的分阶段写作工作流。

它适合每日夜报、热点复盘、财经事件解读这类内容。它的核心不是让 AI 一次性把文章写完，而是把写作拆成几个稳定阶段：先收集素材，再明确主线，再出大纲，再写初稿，再审稿，最后生成发布稿和封面方案。

## 当前这个 skill 是什么

这个 skill 当前服务于「布鲁成长手记」的公众号写作，偏每日财经复盘、市场热点观察和短评型文章。

它会按照固定流程推进：

- `new`: 创建当天写作空间
- `input`: 收集当天看到的新闻、讨论、想法、截图描述
- `specify`: 把混乱素材整理成 3-5 个候选选题
- `outline`: 基于推荐主线生成文章大纲
- `apply`: 写正文初稿
- `review`: 审稿、修订、检查事实和节奏
- `final`: 生成最终可发布稿
- `cover`: 生成公众号封面方案或封面图

创建写作空间后，会生成这些文件：

- `00-input.md`: 原始素材
- `01-specify.md`: 选题整理
- `02-outline.md`: 文章大纲
- `03-draft.md`: 正文初稿
- `04-review.md`: 审稿修订
- `05-final.md`: 最终发布稿
- `06-cover.md`: 封面方案
- `meta.json`: 标题、日期、状态等信息

当前版本的特点：

- 偏公众号夜报和热点复盘
- 适合把新闻、市场异动、讨论和个人判断整理成文章
- 强调先定主线，再写正文
- 不追求一次成稿
- 会区分事实、推测和情绪
- 会检查标题是否准确、是否标题党、是否像新闻拼盘
- 可选参考夜报型文章的结构和节奏，但不复制具体作者口吻

简单说，它现在是一个「布鲁成长手记」专用的每日财经复盘写作流水线。

## 适合改造成什么

这个 skill 的底层结构适合改造成财经公关、企业传播、投资者关系、行业研究、自媒体写作等场景。

尤其适合：

- 财经热点点评
- 公司公告解读
- 市场异动复盘
- 行业政策解读
- 舆情观察
- 投资者沟通材料初稿
- 企业公众号文章
- 高管发言稿前期材料整理
- 周报、日报、内部简报
- 公关回应口径草稿

它最有价值的地方不是某个具体措辞，而是这套流程：

先收素材，不急着写；先定主线，不堆新闻；先分清事实、判断和情绪；再写稿；最后审稿和控风险。

## 如何改造成自己的财经公关 skill

建议不要只把「布鲁成长手记」替换成自己的名字。真正需要改的是内容对象、读者对象、风险边界、标题尺度、审稿标准和输出格式。

### 1. 改 skill 名称和触发名

当前触发名是：

```text
bulu-writing-workflow
```

可以改成：

```text
finance-pr-writing-workflow
ir-content-workflow
market-briefing-workflow
company-comms-writing-workflow
```

同时修改 `SKILL.md` 顶部的 `name`、`description`，以及正文里的触发名。

### 2. 改服务对象

当前服务对象是：

```text
布鲁成长手记
```

可以改成自己的使用场景，例如：

```text
用于某某公司的财经公关、投资者关系和行业内容写作。
```

或者：

```text
用于某某财经账号的每日市场复盘和热点解读。
```

这一部分会影响后续所有写作判断。

### 3. 改默认草稿目录

当前默认草稿根目录是：

```text
/Users/Admin/Documents/bulu
```

可以改成自己的目录，例如：

```text
/Users/你的用户名/Documents/finance-pr
/Users/你的用户名/Documents/ir-content
/Users/你的用户名/Documents/market-briefing
```

如果多人使用，建议把目录写成可配置参数，而不是固定某个人的电脑路径。

### 4. 改文章类型

当前更偏每日夜报和热点复盘。

如果改成财经公关版本，可以加入这些内容类型：

- 公告解读
- 业绩复盘
- 投资者问答
- 舆情回应
- 媒体沟通材料
- 行业趋势短评
- 市场周报
- 企业公众号文章
- 高管观点稿

这样 AI 在整理素材时，不会只按公众号夜报来理解任务。

### 5. 改 `specify` 阶段的判断标准

当前 `specify` 会整理：

- 发生了什么
- 为什么值得写
- 读者关心点
- 用户可能的判断
- 事实、推测、情绪分别是什么
- 一句话白话主线
- 标题方向

财经公关版本建议增加：

- 涉及哪些利益相关方
- 是否涉及监管、资本市场、客户或媒体敏感点
- 哪些内容可以公开表达
- 哪些内容只能内部讨论
- 哪些判断需要弱化
- 哪些数据必须核查来源
- 是否需要准备回应口径
- 是否可能引发误读

### 6. 改 `outline` 阶段的大纲要求

当前大纲偏公众号文章结构。

财经公关版本可以增加：

- 核心信息口径
- 背景解释
- 数据支撑
- 外部关注点
- 潜在质疑
- 回应逻辑
- 风险表达
- 可删减版本
- 对外版和内部版区别

这样它会更像一个公关内容生产工具，而不是单纯写文章工具。

### 7. 改 `review` 阶段的审稿标准

这是最重要的部分。

当前审稿重点包括标题、主线、节奏、事实核查和是否像新闻拼盘。

财经公关版本建议增加：

- 是否存在未经确认的数据
- 是否把传闻写成事实
- 是否有过度承诺
- 是否有监管、合规或信披风险
- 是否容易被媒体断章取义
- 是否对竞品、客户、合作方表达不当
- 是否对股价、业绩、行业趋势作出过度判断
- 是否需要加来源、限定条件或时间范围
- 是否有内部信息外泄风险
- 是否适合对外发布

### 8. 改标题风格

当前标题可以有一点公众号张力。

财经公关版本建议更克制：

```text
标题要准确、清楚、有信息量，但避免夸张、煽动、暗示股价、制造恐慌或作出未经证实的判断。
```

可以允许：

- 解释型
- 观察型
- 复盘型
- 问答型
- 趋势型

建议避免：

- 暴涨暴跌式标题
- 阴谋论式标题
- 绝对化判断
- 监管敏感表述
- 对公司前景的过度承诺

### 9. 改封面要求

当前 `cover` 偏公众号封面。

财经公关版本建议改成：

- 专业
- 清爽
- 稳定
- 不夸张
- 不使用过度情绪化视觉
- 不把敏感判断直接写进图
- 不使用容易被误读的金融符号
- 适合企业公众号、简报或 LinkedIn 风格内容

## 推荐改造后的定位文案

可以把 `SKILL.md` 开头改成这样：

```text
用于财经公关、企业传播、投资者关系和行业内容写作的分阶段工作流。
核心目标不是一次成稿，而是把新闻、公告、市场数据、舆情反馈和内部判断，按固定流程整理成主线清楚、表达克制、风险可控、适合对外发布或内部流转的内容。
```

## 推荐保留的流程

建议保留原来的阶段结构：

```text
new -> input -> specify -> outline -> apply -> review -> final -> cover
```

但把每一阶段的侧重点改成财经公关版：

- `input`: 收集新闻、公告、数据、舆情、内部口径
- `specify`: 判断选题价值、读者关切和风险边界
- `outline`: 搭建文章结构和回应逻辑
- `apply`: 写初稿
- `review`: 做事实核查、口径校准和风险检查
- `final`: 生成对外可用版本
- `cover`: 生成稳健专业的封面方案
